5E SRD Editor - Choosing a Phase

Choosing a Phase and Priority

When you're writing a script, the most important choice you can make is the phase and priority that the script runs at. Certain things need to be done before or after certain times, or they simply won't work. (You can find out when things need to be run by looking at the "How To" Section.)

This document has a list of the existing phases and priorities, giving you an overview of which phases to use for which scripts. Phase ids are specified in italics.

The order in which the phases appear here is the order in which they are executed by Hero Lab.

  1. GlobalTest - Global Testing Phase - This phase can be used for anything that relies on global tags (for example, tags set by rulesets) or controls (for example, checkboxes). It shouldn't be used for anything that depends on anything else.
  2. First - First Phase - This phase can be used for anything that doesn't rely on levels or final attribute values to run. It's basically a catch-all phase for things that don't need to go anywhere else.
  3. PreLevel - Pre-Levels Phase - Can be used by anything that needs to be evaluated before levels are counted and added to the hero.
    For example, you can calculate values in this phase that are then used in later phases, or assign tags that are used in later phases.
  4. Level - Levels Phase - Used internally, typically not used by data file authors.
  5. PostLevel - Post-Levels Phase - Should be used by anything that depends on levels having been counted.
    For example, if your script or tag expression requires checking that the hero is a level 7 Barbarian, you should use this phase.
  6. PreAttr - Pre-Attributes Phase - Should be used by anything that needs to be evaluated before attribute values are totalled up.
    For example, if you want to add 1 to the hero's Strength score, you should do it in this phase or earlier.
  7. Attr - Attributes Phase - Used internally, typically not used by data file authors.
  8. PostAttr - Post-Attributes Phase - Should be used by anything that requires having finalized attributes available.
    For example, if your script needs to know the final value of the hero's Dexterity score, you should run it in this phase.
  9. Final - Final Phase - Should be used by anything that can be run after all other scripts.
    For example, if you want to calculate a field based on the final results of a number of other scripts, that isn't used by anything else, you should use this phase.
  10. Validation - Validation Phase - Used for eval rules that can run after everything is calculated in the previous phases.
  11. Render - Render Phase - This phase should only be used when generating information for display, not for calculating any values.
